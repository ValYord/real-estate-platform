# Sandbox (Docker) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Run the whole agency inside a Docker container (Linux) so agent bash is contained to a mounted workspace, with a repo-scoped GitHub token, and wire the #4a scheduler to run the sandboxed container.

**Architecture:** A `docker/Dockerfile` builds a `python:3.12-slim` image with Node/npm/gh/git + the agency (`pip install -e .`); an entrypoint sets up git/gh auth; a `run-sandbox.sh` launcher mounts only the workspace + a state dir and passes credentials as env; the #4a wrapper calls it. Feasibility was de-risked by a spike (agency + Linux Claude CLI run in-container). This is infra — verification is `docker build` + a real sandboxed run, not pytest.

**Tech Stack:** Docker, bash, Debian slim, Node 20, gh, Python 3.12. Work from `/Users/valeryordanyan/agency`; the agency Python suite is unchanged.

```bash
#!/bin/bash
set -e
# The workspace is a bind mount owned by a different uid; let git operate on it.
git config --global --add safe.directory /workspace
# Configure git to push over https using the GH_TOKEN-authenticated gh credential helper.
gh auth setup-git 2>/dev/null || true
exec agency "$@"
```

- [ ] **Step 3: Create `docker/Dockerfile`** at `~/agency/docker/Dockerfile`:

```dockerfile
FROM python:3.12-slim

# System deps: git, Node 20 (for workspace builds), gh (GitHub CLI).
RUN apt-get update \
    && apt-get install -y --no-install-recommends git curl ca-certificates gnupg \
    && curl -fsSL https://deb.nodesource.com/setup_20.x | bash - \
    && apt-get install -y --no-install-recommends nodejs \
    && mkdir -p -m 755 /etc/apt/keyrings \
    && curl -fsSL https://cli.github.com/packages/githubcli-archive-keyring.gpg \
        -o /etc/apt/keyrings/githubcli-archive-keyring.gpg \
    && chmod go+r /etc/apt/keyrings/githubcli-archive-keyring.gpg \
    && echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" \
        > /etc/apt/sources.list.d/github-cli.list \
    && apt-get update \
    && apt-get install -y --no-install-recommends gh \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app
COPY pyproject.toml README.md ./
COPY src ./src
RUN pip install --no-cache-dir -e .

COPY docker/entrypoint.sh /usr/local/bin/entrypoint.sh
RUN chmod +x /usr/local/bin/entrypoint.sh
ENTRYPOINT ["/usr/local/bin/entrypoint.sh"]
```

(If `~/agency/README.md` does not exist, create a one-line one: `echo "agency" > ~/agency/README.md` — `pip install -e .` reads it via setuptools.)

- [ ] **Step 1: Create `docker/run-sandbox.sh`** at `~/agency/docker/run-sandbox.sh`:

```bash
#!/bin/bash
set -euo pipefail

# Secrets live in the home dir, chmod 600, and `export` their vars:
#   ~/.agency-token     -> export CLAUDE_CODE_OAUTH_TOKEN=...
#   ~/.agency-gh-token  -> export GH_TOKEN=<fine-grained PAT scoped to the repo>
# shellcheck disable=SC1090,SC1091
source "$HOME/.agency-token"
# shellcheck disable=SC1090,SC1091
source "$HOME/.agency-gh-token"

WORKSPACE="/Users/valeryordanyan/real-estate-campony"
mkdir -p "$HOME/agency-state"

exec docker run --rm \
  -v "$WORKSPACE":/workspace \
  -v "$HOME/agency-state":/state \
  -e CLAUDE_CODE_OAUTH_TOKEN \
  -e GH_TOKEN \
  -e GIT_AUTHOR_NAME=ValYord \
  -e GIT_AUTHOR_EMAIL=fitneks.live@gmail.com \
  -e GIT_COMMITTER_NAME=ValYord \
  -e GIT_COMMITTER_EMAIL=fitneks.live@gmail.com \
  agency-sandbox \
  --db /state/agency.db --workspace /workspace "$@"
```

Make it executable:

```bash

```

```markdown
# ADR-012: Sandbox (Docker)

**Status:** Accepted (2026-07-03)

**Context:** The agency runs agent-generated bash directly on the host (ADR-005). Unattended, a
mistaken or prompt-injected command could damage the host or read files outside the project.

**Decision:** Run the whole agency inside a `python:3.12-slim` Docker image (with Node/npm/gh/git
and the agency installed; the Linux Claude Code CLI ships with the pip install — verified by a
spike). Only the workspace (`/workspace`) and a state dir (`/state`, for agency.db) are mounted;
the GitHub credential is a fine-grained PAT scoped to the one repo, passed as `GH_TOKEN`. A
`run-sandbox.sh` launcher and the #4a scheduler run the container.

**Consequences:** Agent bash is contained to the mounted workspace — strong protection against
mistakes and prompt-injection side effects. It is only partial against a truly malicious agent,
which still has network and a repo-scoped token (blast-radius reduction, not perfect isolation).
Network-egress filtering, token longevity (#5b), and cloud hosting (#4b) are out of scope. Headless
token auth in-container is verified before adoption (see docker/README.md); if it fails, mount the
host Claude config as a fallback.
```
